HTML, CSS and DOM Analogy
Imagine a highly decorated wedding cake. Tp make a wedding cake you have to start with a plain old cake, the HTML is the plain old cake its the "structure" of the wedding cake it holds all the decoration. Now the CSS is the decoration, there are diffrent elements just like cake decoration, its got marzipan, ribbons, edible flowers its what makes the cake beautiful. the HTML is the plain old cake of a website and the CSS is what makes it beautiful. The DOM is like an instruction video of the cake makling process so that you can see how each step of the decoration and cake could be done but not neccesarily EXACTLY how you did it in yours.
What is boxifying a design
Boxifying is a way of braking down a web page layout for easier building, by thinking of each element in the page as a "box" and the page as an arrangement of those boxes
Explain the box model
this is a model used to calculate the layout of your web site, it factors in 4 diffrent areas
  • The Margin, this is a clear area around the outside of your page leaving space between the sides of the web browser and your design. it displays nothing
  • The Border, this is inside the margin and is a container for your content
  • The Padding, this is a space between your boarder and content, it generally makes for a tidy website but not com pulsory, is is transparent.
  • The Content, this is your work. The text, images ect that make up your website

using this model it is easier to calculate how your layout will look and function, and how much space to assign to different areas
