The time I got blocked on a simple problem

What was the problem?
When making the racer game, I really had no idea how to make the HTML page "talk" with the Javascript. I knew about linking the JS file with the script tag but then how to make the Javascript react to keypresses on the browser through the HTML file.
What problem solving techniques did you use?
I tried searching the internet for blogs, articals and tutorials on Javascript-HTML basics anything that would give me a bit more understanding of thew way the two files function together.
How did you feel throughout the process?
I looked at the ultimate requirements and started braking it down into simpler demands and built it up. So I began by just making the pageload call a function and print a word to the console, then I researched how to make that function call happen on a key press, and added bits in untill I had a good idea of how to work the code.
What did I learn?
Psudocode is your best friend. If your task seems unmanageable brake it down into smaller tasks untill each one is manageable.

Explain functions

Explain .map()
The map function cycles through items in an array, the function is usually used to make a change to the items in the array so the function usually assigns the array items to another array.

Explain .filter()
The filter function cycles through the items in an array and picks out some of the elements and assigns them into another array. the argumnets passed in decide what items are picked out

Explain .reduce()
The reduce function cycles through the items in an array, and assigns them into a single item varible. To put it into a single item varible it cant be an array anymore, so it is usually used to concatenate strings or sum up numbers.
